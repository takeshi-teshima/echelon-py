Examples

>>> h = np.array([1, 2, 3, 4, 3, 4, 5, 4, 3, 2, 3, 4, 5, 6, 5, 6, 7, 6, 5, 4, 3, 2, 1, 2, 1])
>>> W = np.zeros((25, 25), dtype='int8')
>>> W[np.eye(len(W), k=1, dtype='bool')] = 1
>>> W[np.eye(len(W), k=-1, dtype='bool')] = 1
>>> from echelon.oracle import NdarrayEchelonOracle
>>> oracle = NdarrayEchelonOracle(h, W)
>>> find_peak_echelons(oracle)
[[16, 17, 15], [13], [6, 5, 7], [3], [23]]

Notes

After finding the initial peaktop in each iteration, the algorithm takes the “set” of argmax indices, i.e., it considers the contour set in the neighborhood simultaneously as candidates.

Algorithm

  1. We repeat the following until we run out of all unsearched indices.

    1. Select one maximum point (summit, \(s\)) and the max value (summit value, \(v\)) among the unsearched indices. That is, \(s \in \mathrm{argmax}_{i \in \mathrm{unsearched}}\mathrm{Data}_i\) and \(v = \mathrm{Data}_s\).

    2. Initialize \(\mathrm{peak}=\{s\}\).

    3. We now run the following extension loop:

      We consider whether to extend \(\mathrm{peak}\) to newly include \(S := \mathrm{argmax}_{i \in N} \{\mathrm{Data}_i\}\) where \(N := \mathrm{nb}(\mathrm{peak})\), i.e., the maximizers among the neighbors of \(\mathrm{peak}\). Here, argmax is understood as a set of indices.

      • If \(S\) contains no local minimum (i.e., if \(\mathrm{Data}_{i} \geq \max_{i \in \mathrm{nb}(N\cup S)}\mathrm{Data}_{i}\) where \(i \in S\)), then we extend the peak set: \(\mathrm{peak}\gets \mathrm{peak}\cup S\).

      • Otherwise, we halt the extension loop, flag all elements of \(\mathrm{peak}\) as searched, register \(\mathrm{peak}\) as a found peak echelon, and go back to the top of the loop.

  2. After the loop is halted, we have the set of found \(\mathrm{peak}\)’s (which may or may not jointly contain all indices. Those indices that do not belong to a peak echelon will belong to a foundation echelon later.).
