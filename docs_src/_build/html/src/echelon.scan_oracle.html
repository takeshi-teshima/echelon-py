echelon.scan_oracle module

This module contains the oracles used to run the echelon scan for hot-spot detection. The oracles provide the scores for the hot-spot detection.

class ApproxBinomialScanOracleBase(all_total_count, all_marked_count)[source]

Bases: echelon.scan_oracle.ScanOracleBase

Notes

Consider \(K\) regions and let us denote \([K] := \{1, 2, \ldots, K\}\). For each region \(k \in [K]\), assume \(c_k \sim \mathrm{Bi}(\ heta_k, n_k)\).

Consider a region \(Z \subset [K]\). We assign a score \(s\) that indicates how “hot-spot-ish” the region \(Z\) is.

Let us assume that \((\theta_k)_{k\in Z}\) and \((\theta_k)_{k\in Z^\mathrm{c}}\) are constants, \(\theta_Z\) and \(\theta_{Z^\mathrm{c}}\), respectively. We compare the maximum likelihood under \(\theta_Z = \theta_{Z^\mathrm{c}}\) and \(\theta_Z > \theta_{Z^\mathrm{c}}\). Denote \(c_Z := \sum_{k \in Z} c_k\), \(n_Z := \sum_{k \in Z} n_k\), \(n_{Z^{\mathrm{c}}} := \sum_{k \in Z^{\mathrm{c}}} n_k\), and \(n_{Z^{\mathrm{c}}} := \sum_{k \in Z^{\mathrm{c}}} n_k\).

Now, by the reproductive property, \(c_Z \sim \mathrm{Bi}(\theta_Z, n_Z)\) where \(\theta_Z = \sum_{k \in Z} \theta_k\). Similarly for \(c_{Z^\mathrm{c}}\). Since the maximum likelihood of \(c \sim \mathrm{Bi}(\theta, n)\) is \(\mathrm{Bi}(\frac{c}{n}, n)(c)\) (by abuse of notation, we denote the density of a distribution by the name of the distribution).

Now, when \(n >> c\), we have \(\mathrm{Bi}(\theta, n)(c) \simeq (\theta)^c\).

With this approximation, the score for the case that \(\theta_Z > \theta_{Z^\mathrm{c}}\) is \(\log (\frac{c_Z}{n_Z})^{c_Z} + \log (\frac{c_{Z^\mathrm{c}}}{n_{Z^\mathrm{c}}})^{c_{Z^\mathrm{c}}}\).

Similarly, the score for the case that \(\theta_Z = \theta_{Z^\mathrm{c}}\) is \(\log (\frac{c_{[K]}}{n_{[K]}})^{c_{[K]}}\).

Thus, we use \(\log (\frac{c_Z}{n_Z})^{c_Z} + \log (\frac{c_{Z^\mathrm{c}}}{n_{Z^\mathrm{c}}})^{c_{Z^\mathrm{c}}} - \log (\frac{c_{[K]}}{n_{[K]}})^{c_{[K]}}\) as the score.

class PoissonScanOracleBase(all_count, total_index_len)[source]

Bases: echelon.scan_oracle.ScanOracleBase

Notes

Consider \(K\) regions and let us denote \([K] := \{1, 2, \ldots, K\}\). For each region \(k \in [K]\), assume \(c_k \sim \mathrm{Po}(\lambda_k)\) and that \(\{c_k\}_{k \in [K]}\) are independent.

First, note that \(\mathrm{Po}(c; \lambda) := \frac{\lambda^c e^{-\lambda}}{c!}\) is maximized by \(\lambda = c\). Thus, the maximum likelihood given \(c\) is \(\frac{c^c e^{-c}}{c!}\).

Consider a region \(Z \subset [K]\). We assign a score \(s\) that indicates how “hot-spot-ish” the region \(Z\) is.

By the reproductive property, \(c_Z := \sum_{k \in Z} c_k \sim \mathrm{Po}(\lambda_Z)\) where \(\lambda_Z = \sum_{k \in Z} \lambda_k\). Similarly for \(Z^\mathrm{c}\).

We compare the maximum likelihood under \(\lambda_Z = \lambda_{Z^\mathrm{c}}\) and \(\lambda_Z > \lambda_{Z^\mathrm{c}}\). The maximum log likelihood in the first case is \(\ell_0 := \log \frac{c_{[K]}^{c_{[K]}} e^{-{c_{[K]}}}}{c_{[K]}!}\) That for the second case is \(\ell_1 := \log \frac{c_{Z}^{c_{Z}} e^{-{c_{Z}}}}{c_{Z}!} + \log \frac{c_{Z^\mathrm{c}}^{c_{Z^\mathrm{c}}} e^{-{c_{Z^\mathrm{c}}}}}{c_{Z^\mathrm{c}}!}\)

Thus, we use \((\ell_1 - \ell_0) [[c_Z > c_{Z^\mathrm{c}}]]\) as the score.
