>>> find_foundation_echelons(oracle, peak_echelons)
[[12, 14, 18, 19, 11, 10, 20], [2, 4, 8], [1, 9, 21], [0, 22, 24]]

Notes

The algorithm is as follows. In this algorithm, we keep track of the list of families. Each “family” is a set of indices, and the initial set of families consist of the peak echelons. In the latter steps, the set of families grow.

We define \(\mathrm{TangentFamily}(\mathrm{indices}) := \{F \in \mathrm{families} : F \cap \mathrm{nb}(\mathrm{indices}) ≠ \emptyset\}\). We also define \(\mathrm{DataMax}(\mathrm{indices}) := \max_{i \in \mathrm{indices}}\mathrm{Data}_i\) and \(\mathrm{DataArgMax}(\mathrm{indices}) := \mathrm{argmax}_{i \in \mathrm{indices}}\mathrm{Data}_i\)

Algorithm

  1. Mark all indices in Peak Echelons as searched.

  2. We repeat the following until we run out of all unsearched indices.

    1. Select one maximum point (foundation-top, \(t\)) and the max value (\(v\)) among the unsearched indices. That is, \(t \in \mathrm{DataArgMax}(\mathrm{unsearched})\) and \(v = \mathrm{DataMax}(\mathrm{unsearched})\).

    2. Initialize \(\mathrm{foundation}=\{t\}\).

    3. Initialize the set of families \(\mathrm{families} = \{\mathrm{peak}'s\}\).

    4. Initialize the temporary family \(\mathrm{tempFamily} = \mathrm{foundation}\).

    5. We now run the following extension loop, consisting of two steps: family extension and neighbor extension.

      1. Family extension.

        1. Pop out the set of tangent families:

          • \(\mathcal{F} = \mathrm{TangentFamily}(\mathrm{tempFamily})\).

          • \(\mathrm{families} \gets \mathrm{families} \setminus \mathcal{F}\).

        2. Merge the tangent families as \(\mathrm{tempFamily} \gets \mathrm{tempFamily} \cup (\cup \mathcal{F})\).

      2. Neighbor extension. We decide whether to extend the set of families to neighbors.

        1. Get the set of neighbors \(N := \mathrm{nb}(\mathrm{tempFamily})\).

        2. Find \(A := \mathrm{DataArgMax}(N)\).

          • If the maximum value \(\mathrm{Data}_i\) (\(i \in A\)) equals :math`v`, then we regard \(A\) to be as qualified as \(t\) to be part of \(\mathrm{foundation}\), and therefore

            • Add \(A\) to the foundation: \(\mathrm{foundation} \gets \mathrm{foundation} \cup (A \cap \mathrm{unsearched})\).

            • Add \(A\) to the temporary family: \(\mathrm{tempFamily} \gets \mathrm{tempFamily} \cup (A \cap \mathrm{unsearched})\).

            • Mark \(A\) as searched.

          • If elements of \(A\) contains no local minima, i.e., if \(\mathrm{DataMax}(A\cup \mathrm{tempFamily}) \geq \mathrm{DataMax}(\mathrm{nb}(A\cup \mathrm{tempFamily}))\), then

            • Add \(A\) to the foundation: \(\mathrm{foundation} \gets \mathrm{foundation} \cup (A \cap \mathrm{unsearched})\).

            • Add \(A\) to the temporary family: \(\mathrm{tempFamily} \gets \mathrm{tempFamily} \cup (A \cap \mathrm{unsearched})\).

            • Mark \(A\) as searched.

          • Otherwise, we halt the iterative loop, register \(\mathrm{foundation}\) as a foundation echelon, add \(\mathrm{tempFamily}\) to the set \(\mathrm{families}\) as an element, and go back to the top of the loop

  3. After the loop is halted, we have the set of found \(\mathrm{foundation}\)’s.

The set of foundation echelons and the set of peak echelons are mutually exclusive and collectively exhaustive.
