Welcome to echelon-py’s documentation!

Installation

$ pip install echelon-py

Usage

Various examples are implemented in the echelon-py-examples repository.

The API modules

  • echelon.api.dataframe_api module

  • echelon.api.ndarray_api module

The API modules provide easy interface to access the integrated functionalities of the package.

In case these APIs do not match your needs, you can also combine the algorithms separately. To learn how to combine the different modules in this package, you can look into the API code.

Some of the functionalities that are common to the APIs are implemented in the base module (echelon.api.base module).

The Contributed modules

The “contributed” modules provide utility functionalities to handle external packages, various data types, etc. Currently, the following modules are available:

  • echelon.contrib.geo module

The module contents

These modules are called from the API modules.

  • Algorithms: implemented in the following modules.

    • echelon.algorithms.find_peak_echelons module
    • echelon.algorithms.find_foundation_echelons module
    • echelon.algorithms.find_echelon_clusters module
    • echelon.algorithms.find_echelon_hierarchy module
    • echelon.algorithms.find_echelon_hotspots module

  • Oracles: abstract the access to the data. For custom input data structures, you can implement a new oracle (inheriting EchelonOracleBase) using the implemented classes such as DataFrameEchelonOracle as examples.

    • echelon.oracle module
    • echelon.scan_oracle module

  • Typing: Some types for type hinting are defined in the following module:

    echelon.typing module

Optional Dependencies

  • scipy to enable dendrogram.

  • anytree to store and visualize the echelon hierarchical structure.

Call for Contributions

The algorithms have not been optimized for efficiency (sorry), and there are probably obvious ways to improve the performance.

  • If you encounter a performance issue, please file it on GitHub.

  • If you notice an edge case, please file a test case issue on GitHub.

  • If you are kind enough to refactor the algorithms for efficiency, please feel free to open an issue and submit a pull request on GitHub.
