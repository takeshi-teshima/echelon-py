echelon.algorithms.find_echelon_hotspots module

find_echelon_hotspots(scan_oracle, tree_root, echelons, original_oracle)[source]

Notes

This method constructs the candidates of the hot spot \(Z\) using the information from the echelon hierarchy. Algorithm

  • Let \(\{E_k\}_{k=1}^K\) be the echelons sorted (in descending order) based on the maximum observation values of the indices in the echelon (used for constructing the echelons).

  • Let \(E_k = \{e_{k,j}\}_{j=1}^{n_k}\) be the indices in each echelon sorted (in descending order) based on the original observation values.

  • We define the hot-spot candidates \(Z_{kj} := (\cup_{k' \in \mathrm{descendants}(k)} E_{k'}) \cup \{e_{k,1}, \ldots, e_{k,j}\}.\) That is, we accumulate the elements in \(E_k\) up to the index \(j\), as well as all of its descendants.

  • We score the candidates based on the binomial scan score.
